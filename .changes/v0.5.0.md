## [v0.5.0] - 2026-03-29

### Added

- Declarative hook filters — scope hooks by payload fields without script-level filtering
- TUI mouse support: click-to-select, tab switching, scroll wheel in all views
- orch_dispatch skip_handoff flag suppresses per-dispatch auto-handoff chains
- orch_wait_merge MCP tool — block until a merge operation reaches terminal status with progress notifications
- orch_commit MCP tool — commit worktree changes for MCP-only agents
- Mouse click support for log viewer tab bar (Input / Timeline / Output)
- Config validation warnings for dangerous backend_args flags and sensitive env var overrides
- Best-effort secret redaction in execution logs and output previews
- Ops tab: dedicated Failures section after Running, section reorder to match operator priority, and error_detail in failed row summaries
- orch_list_agents now returns per-agent max_concurrent/active/queued/available and global capacity totals
- SEC-6: Queue depth guard (max_queued_executions), orch_abandon kills subprocesses, orch_abandon_batch tool

### Changed

- orch_close no longer auto-merges; completed worktree threads require explicit orch_merge before close
- Restructure 'compas wait' as nested subcommands: 'compas wait message' and 'compas wait merge'; removes 'compas wait-merge'
- orch_merge now accepts Active threads, enabling the merge-before-close workflow
- Update all documentation to use new 'compas wait message' / 'compas wait merge' subcommand syntax
- Rewrite execution detail overlay with error-first card and tabs layout
- Filter superseded failed merge ops from TUI merge queue when a later success exists for the same (thread_id, target_branch) pair
- Reorder execution detail tabs to Input-first; render Input tab content as markdown
- Output tab in execution detail shows agent narrative text instead of raw JSONL protocol events in humanized mode
- Execution detail: fix markdown rendering (standalone mode, blank-above-headings, themed inline code, dynamic rules, stronger code blocks), apply markdown to Output tab, reorder tabs to Input | Output | Timeline
- Enforce restrictive file permissions (0700/0600) on state directory and files (Unix)
- Ops tab: running items always show progress summary sub-line; footer adapts to terminal width in three tiers (<90, 90-119, ≥120 cols)
- Require explicit safety_mode: auto_approve on built-in backend agents
- Dashboard Agents tab UX: per-agent health dot from circuit breaker state, Enter drills into agent executions, compact card headers for 80-col terminals, relative timestamps on execution summaries, help overlay includes Agents and Settings sections
- History tab UX: drop Thread ID/Prov columns for 80-col Error visibility, batch headers as full-width dividers with completion progress, context-sensitive footer hints, elapsed time for running executions, remove blank separator rows
- Settings tab: scrollable content, keyboard navigation (j/k/g/G), r to refresh schedules, consistent label alignment, 80-col schedule table, empty state message
- Clarify ticket-session authority and active queue guidance
- Pre-commit hook now runs ticket reconcile on code commits as the authoritative session consistency check
- orch_list_agents now includes handoff routing metadata (handoff_to, await_chain_recommended, max_chain_depth) and orch_dispatch next_step hints are target-agent-aware

### Fixed

- Detect no-op merges and uncommitted worktree changes before merge execution
- Merge commits now use repo git identity instead of hardcoded compas@localhost; commit message sourced from thread summary
- Execution detail overlay now opens on Input tab by default
- Accept string-encoded numeric params in orch_wait, orch_wait_merge, and orch_read_log parameters
- orch_dispatch auto-reopens terminal threads instead of silently dropping messages
